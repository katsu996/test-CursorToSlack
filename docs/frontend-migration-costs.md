# フロントエンド（Vite + React + shadcn/ui 本体）移行の判断材料

現状は **ビルドなしの静的 HTML**（`docs/index.html` / `level-stats.html`）と **共通 CSS**（`docs/assets/k-original-ui.css`）で運用しています。表・フィルタがさらに複雑になる場合の **おおよそのトレードオフ**です（日付ベースの見積もりはしません）。

| 観点 | 現状（静的 HTML） | 移行後（Vite + React + shadcn 本体） |
|------|-------------------|--------------------------------------|
| 依存関係 | なし（GitHub Pages がそのまま配信） | `node_modules`、ロックファイル、脆弱性対応 |
| CI | Python のみ | `npm ci` + build + artifact サイズ増 |
| 開発体験 | 手書き JS・重複に弱い | コンポーネント分割・型付け・Storybook 等 |
| デザイン | CSS で shadcn「風」 | デザインシステムに忠実 |
| 長期保守 | UI 変更の横断が手作業 | リファクタしやすいがツールチェーン維持が必要 |

**移行を検討するトリガー例:** 表以外の画面が増える・状態管理が複雑化する・複数人でフロントを触る頻度が上がる、など。

**移行しない利点:** リポジトリが軽く、Actions が単純で、Pages の設定変更が少ない。

## バンドル・型付けを入れる場合の追加コスト（現状維持との比較）

次は「Vite/React まで行かず、**バンドル（esbuild 等）と TypeScript だけ**」を足す中間案も含めた整理です。

| 項目 | 静的のみ（現状） | バンドル + TS（中間） | Vite + React + shadcn 本体 |
|------|------------------|----------------------|----------------------------|
| **ツール** | なし | `package.json`、ビルド 1 本（例: esbuild）、`tsc --noEmit` または `esbuild` の型チェック連携 | Vite、React、eslint/prettier、コンポーネント層 |
| **CI** | `pages.yml` は Python のみ | `npm ci` と `npm run build` を **build ジョブ**に追加し、`docs/` へ出力した成果物をそのまま deploy artifact に載せる必要がある | 同左＋テスト・Lint の実行時間増 |
| **Pages** | `docs/` をそのまま公開 | ビルド成果物を `docs/` または `docs/dist/` に出す運用を決める（**公開ルートとビルド出力の対応**を README / `ci-github-pages-workflow.md` に書く） | 同左 |
| **型付け** | なし（実行時のみ気づく） | 表スキーマ・`meta.pages_ui` 連携を型で縛れるが、**Python 生成 JSON と二重定義**になりやすい（生成物から型を起こすなら別途スクリプト） | コンポーネント props とストアで恩恵大、保守コストも最大 |
| **デバッグ** | ブラウザのソースがそのまま | source map、ビルドステップの切り分け | HMR、ただし本番ビルド差分の調査は重い |

**判断の目安（2026 時点の方針メモ）**

- **現状維持（静的のみ）**でよい条件: 変更は `pages-index-*.js` と `pages_ui_config.json` の範囲に収まり、ユニットテスト（`check_browser_rows_pages_ui` 等）で回帰を拾えている。
- **中間案**を検討する条件: 同一ロジックのコピペが増え、手書き JS のバグが目立ち始めたが、まだ画面数・状態は少ない。
- **フル移行**を検討する条件: 画面・ルート・共有状態が増え、CSS だけでは構造が追いづらいとチームで合意したとき。

このファイルは上記のように **コストと「静的配信のみ」を続ける理由**を短く残すためのものです。CI や `docs/` の実際の手順は **[ci-github-pages-workflow.md](./ci-github-pages-workflow.md)** とルート **README.md** を正としてください。
