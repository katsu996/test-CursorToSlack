# =============================================================================
# 初回だけ: このファイルをコピーして secrets ファイルを作る
# =============================================================================
# 1) このファイル全体を、次と同じフォルダにコピーする:
#      upload-songdata-github-release.ps1
#      upload-songdata-github-release.bat
# 2) コピー先のファイル名を、次の名前に必ずリネームする:
#      upload-songdata-github-release.secrets.txt
# 3) 下の「1 行目」「2 行目」を、このコメントのルールどおりに編集する。
# 4) Windows のメモ帳で保存するとき:
#      「名前を付けて保存」→ 文字コード: UTF-8（BOM 付きでも可）
#      「Unicode（UTF-16）」は使わない（1 行目のトークンが壊れます）。
# =============================================================================

# --- 1 行目（必須）: Personal Access Token（PAT）---------------------------
# トークンは GitHub で作成した直後に一度だけ全文が表示されます。
# よくある先頭: ghp_（classic） / github_pat_（fine-grained） / ghs_ / gho_ など
#
# ↓↓↓ 次の行を編集（サンプル ghp_REPLACE_ME を消して本物の PAT のみにする）↓↓↓
ghp_REPLACE_ME

# --- 2 行目（必須）: アップロード先のリポジトリ ---------------------------
# 形式: 所有者/リポジトリ名   例: katsu996/test-CursorToSlack
# Release を作る GitHub リポジトリと一致させる。
#
# ↓↓↓ 次の行を編集↓↓↓
katsu996/test-CursorToSlack
